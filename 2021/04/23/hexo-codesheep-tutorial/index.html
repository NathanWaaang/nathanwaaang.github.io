Hexo | CodeSheep 博客搭建教程整理

发表于 2021-04-23 更新于 2021-05-05 阅读次数： Valine：

本文为 Bilibili up 主 CodeSheep 的 手把手教你从0开始搭建自己的个人博客 |无坑版视频教程| hexo 教程文字整理，视频中搭建环境为 Mac OS，本文则整理了 Windows 环境下如何使用 Hexo 框架搭建博客。此处附上 CodeSheep 的个人主页。

Why Hexo?

  1. Hexo 作为最近比较火的博客搭建框架，上手容易，且社区活跃，在网络上相关的知识较多。
  2. 使用 Hexo 搭建静态博客，操作简单，容易维护。

环境搭建

  1. Node.js，包含 Node.js 本身和 npm 包管理器，是搭建 Hexo 的必要内容
  2. Git，一些命令的执行依赖 Git，有博客说有的命令需要在 Git Bash 下执行，但本次博客搭建全程使用以管理员身份运行的命令提示符。
  3. 安装完成后，使用管理员身份进入命令行
  4. 使用 node -v 和 npm -v 命令查看是否安装成功，若出现版本信息则说明 Node 安装成功
  5. 国内 npm 下载很慢，安装 cnpm（淘宝）。使用命令 npm install -g cnpm --registry=https://registry.npm.taobao.org
  6. npm 下载完成后输入命令 cnpm，若出现用户操作手册说明安装成功
  7. 使用 cnpm 安装 hexo 框架，运行命令 cnpm install -g hexo-cli（其中 -g 代表全局安装）
  8. 使用 hexo -v 查看 hexo 是否安装成功，可以看到其中有 node 版本，说明 hexo 是基于 node 的

使用 Hexo 启动博客

  1. 新建文件夹用以储存本地博客文件，任何路径都可以，本次实验在 D:\Works\Blog。本次实验的所有操作均在该文件夹内进行，若某一部有问题或者不满意均可直接删除文件夹
  2. cd 到该文件夹下，本次实验为 cd D:\Works\Blog，使用 hexo init 命令进行初始化。出现 INFO Start blogging with Hexo! 说明安装成功
  3. 使用 hexo s 启动 hexo，可以发现在 http://localhost:4000 启动（一般用来预览文章）
  4. 访问 http://localhost:4000，可以发现 博客已经存在，并默认创建了 Hello World 文章

使用 Hexo 新建博客

  1. 使用 hexo n <filename> 或 hexo new <filename> 新建博客，本次为 hexo new "FirstTry"。在 D:\Works\Blog\source_posts 目录下已经新建了 FirstTry.md 文件。可以使用 Typora 等修改博客
  2. 使用 hexo clean 清除缓存文件和已生成的静态文件
  3. 使用 hexo generate 生成静态文件
  4. 使用 hexo s 启动，发现 FirstTry 已经上传到了博客中

将博客部署到 Github 上

  1. 可以发现前面的博客都是启动在本地，若想其他人访问则需部署到远端
  2. 本次推荐使用 Github，可以通过 Github 地址访问博客
  3. 登录 Github，新建仓库。用户部署个人博客的 Github 仓库命名必须符合特定要求，必须为 <username>.github.io
  4. 在本地目录下装 Git 部署的插件。在博客目录路径下使用命令 cnpm install --save hexo-deployer-git
  5. 设置 _config.yaml 文件（本次实验为 D:\Works\Blog_config.yaml）在最下面的 Deployment 进行配置
    1. 将 type 一行改为 type: git（type 后面本来有引号，注意去掉）
    2. 在 type 下一行对齐写入 repo: <your_repo_addr>。<your_repo_addr> 可以直接在 Github 相应的仓库中点击 Code 获取 HTTPS 地址。
    3. 在 repo 下一行对齐写入 branch: master（默认也是 master，这里相当于显式写明）
    4. 注意上面的 type、repo、branch 冒号后均有一个空格
    5. 使用 hexo d 命令进行部署，在执行命令后会弹出 Github 登录窗口（可能需要挂 vpn，本次使用校园网）。相当于往远端推本地文件
    6. 此时再次访问 Github 的仓库页面，发现不再是空仓库，而变为博客的相关内容
    7. 直接在浏览器里访问网站 <username>.github.io，即可访问博客。

为博客自定义主题

  1. 可以使用别人的主题，也可以自定义。本次 up 主推荐的为 github.com/litten/hexo-theme-yilia，本次实验尝试改为这个主题
  2. 将主题克隆到本地博客的 themes 文件夹下。本次实验为 git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia
  3. 配置主题到博客中，同样设置 _config.yaml 文件（本次实验为 D:\Works\Blog_config.yaml）找到 theme 这一行，将默认的 landscape 改为 yilia
  4. 重新执行 hexo clean、hexo generate、hexo s，此时可以看到本地主题已经更换
  5. 重新执行 hexo d，将本地文件推到远端

本文总结

  1. 环境准备：Git、Node，使用 Node 安装 Hexo
  2. 使用 Hexo 命令，初始化博客、新建博客、部署博客，并更换主题
